iQube Architecture

How the browser, encryption service, IPFS, blockchain, and Supabase work together.

Mint Flow — Step by Step

1

User fills the wizard

The browser collects public details (name, type, visibility) and private fields (credentials, PII, etc.). Nothing leaves the browser yet.

Browser only
2

Encrypt private fields

The app sends the private fields to the encryption service (Supabase Edge Function or your existing backend). The server generates a random AES‑256 key + IV, encrypts the data, stores the key in Supabase, and returns only the encrypted blob.

Key stored in Supabase Encrypted blob returned to browser
3

Upload metadata to IPFS

The browser builds a JSON document containing the public attributes and the encrypted blob, then uploads it to Pinata/IPFS. Returns a content‑addressed URL.

JSON with encrypted data → IPFS
4

Mint on‑chain

The browser asks MetaMask to call mintQube(to, ipfsUri) on your iQubeNFT contract. The user pays gas in POL. The contract mints an ERC‑721 token and stores the IPFS URI.

Ownership + IPFS URI on Polygon
5

Decrypt (later, owner only)

When the owner wants their data back, the app calls a Supabase Edge Function. The function checks ownerOf(tokenId) on‑chain. If the caller matches, it retrieves the key from Supabase and decrypts the IPFS blob.

Key never leaves Supabase Ownership verified on‑chain

What Each System Stores

Polygon (Amoy)

Public ledger
  • Token ownership (who owns which iQube)
  • IPFS URI per token (pointer to metadata)
  • Transfer history

Pinata / IPFS

Encrypted content storage
  • Public metadata (name, type, risk score)
  • Encrypted private payload (ciphertext only)
  • No keys, no plaintext

Supabase

Key vault + app database
  • AES encryption keys (per tokenId)
  • Decrypt API (checks on‑chain ownership first)
  • User accounts, sharing rules, audit logs

Browser App

User interface
  • Wizard UI (React + Vite)
  • MetaMask wallet connection
  • Calls encrypt, upload, and mint APIs
  • Never stores keys or plaintext after mint

Key Insight

Why three systems?

Chain = "who owns what" + "where is it" (trust + transferability).
IPFS = "what it is" in encrypted form (cheap, immutable storage).
Supabase = "who can decrypt it" (keys never on‑chain, never in the browser).
